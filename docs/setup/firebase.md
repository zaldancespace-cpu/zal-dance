# Настройка Firebase для «ЗАЛ»

## 1. Создать проект

1. Открой Firebase Console.
2. Создай новый проект для танцевального зала.
3. Добавь Web App.
4. Скопируй значения Firebase config в локальный `.env` по примеру `.env.example`.

## 2. Включить Firestore

1. Открой Firestore Database.
2. Создай базу в production mode.
3. Опубликуй правила из `firestore.rules`.

## 3. Включить вход администратора

1. Открой Authentication → Sign-in method.
2. Включи Email/Password.
3. Создай пользователя администратора.
4. Скопируй `uid` администратора.
5. Создай документ `adminUsers/{uid}`:

```json
{
  "email": "admin@example.com",
  "role": "admin"
}
```

## 4. Коллекции

Backend будет создавать и обновлять:

- `bookings` — полные бронирования с контактами клиента, доступны только админам.
- `blockedSlots` — ручные блокировки, доступны только админам.
- `paymentEvents` — журнал платежных событий, закрыт для клиента.
- `publicAvailability` — обезличенная публичная доступность по датам.

Frontend без входа читает только `publicAvailability`, чтобы календарь не раскрывал персональные данные клиентов.
